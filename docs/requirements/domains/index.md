# Domains

Business and capability domains that define bounded contexts within the system. Domain pages document key concepts, relationships, and which applications or features fall within each domain.

## Example

- [DOMAIN-EXAMPLE-001 — Example Domain: {DOMAIN-NAME}](DOMAIN-EXAMPLE-001.md)

## ID Pattern

Domains follow the pattern `DOMAIN-<NNN>` where `<NNN>` is a zero-padded sequence number (e.g. `DOMAIN-001`).
